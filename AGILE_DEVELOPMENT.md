## 🔄 Git 工作流程

### 分支策略

| 分支 | 用途 | 说明 |
|------|------|------|
| `main` | 生产环境 | 只接受通过测试的功能合并 |
| `dev` | 日常开发 | 开发工作在此分支进行 |

### 开发流程

```bash
# 1. 切换到开发分支
git checkout dev

# 2. 创建功能分支（可选）
git checkout -b feature/新功能名

# 3. 开发并测试...

# 4. 提交代码
git add <files>
git commit -m "feat: 描述改动"
git push origin dev

# 5. 创建 Pull Request
# 在 GitHub 上创建 PR 从 dev -> main
```

### Commit Message 规范

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加股价预警功能` |
| `fix` | Bug 修复 | `fix: 修复新闻抓取超时问题` |
| `docs` | 文档更新 | `docs: 更新 README 使用说明` |
| `refactor` | 重构 | `refactor: 优化数据获取逻辑` |
| `chore` | 配置/工具 | `chore: 清理冗余文件` |

## 🔧 配置管理

### 飞书配置 (feishu_config.json)

**创建时间：** 2026-03-11  
**目的：** 集中管理飞书推送凭证，避免硬编码

**配置内容：**
```json
{
  "feishu": {
    "enabled": true,
    "user_id": "YOUR_USER_ID",
    "app_id": "YOUR_APP_ID",
    "app_secret": "YOUR_APP_SECRET",
    "retry_times": 3,
    "retry_delay_seconds": 2
  },
  "notification_settings": {
    "stock_push": {"enabled": true, "format": "single"},
    "price_alert": {"enabled": true, "threshold_up": 5.0, "threshold_down": -5.0},
    "prediction_push": {"enabled": true, "schedule": "15:30"}
  }
}
```

**安全保护：**
- ✅ 已加入 `.gitignore`，不会被提交到 Git
- ✅ 提供模板文件 `feishu_config.example.json`
- ✅ 所有脚本从配置文件读取凭证

**修改的脚本：**
- `multi_stocks_monitor.py` - 从配置文件加载
- `prediction_push.py` - 从配置文件加载
- `price_alert_monitor.py` - 从配置文件加载
